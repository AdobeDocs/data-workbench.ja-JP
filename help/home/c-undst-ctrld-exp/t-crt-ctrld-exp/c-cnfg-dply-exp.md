# テストの設定と導入{#configuring-and-deploying-the-experiment}

目的、仮説、実験の詳細を定義し、テストコンテンツを作成したら、制御された実験を導入するようにSensorを設定する必要があります。

## テスト設定ファイルの設定 {#section-037fe7dea9c94aee9cdc354dafdb7c03}

テストを設定するには、アドビが提供する（デフォルトで名前が付けられている）テスト設定スプレッドシートを完成さ [!DNL TestExperiment.xls] せる必要があります。 このファイルは、 [!DNL Sensor] テストを実行するように設定され、ExpFileパラメータの変更で指定したExcelバー [ジョンのテキストファイルです](../../../home/c-undst-ctrld-exp/t-en-ctrld-exp/c-mod-expfile-prm.md#concept-25232b386a654870becc789d4f1fcc28)。

このファイルには複数の実験に関する情報を含めることができます。複数の実験は同じ時間や異なる時間に実行され、異なるグループや割合を使用できますが、これらの実験はどのような方法でも相関しません。

この時点で実行するように設定されたファイル内の各実験のグループにユーザーが配置されます。

>[!NOTE]
>
>各実験は他の実験とは独立している。 ある実験に加えた変更は他の実験に影響を与えず、訪問者が複数の実験を行う場合もありますが、結果は相互に関連しません。 複数の実験の変化の間に相関関係があると考える場合は、これらの変化を一緒にテストする新しい実験を作成する必要があります。

**テストを設定するには**

テストの開始前にこのファイルを完成させ、テストの実行中に情報を変更しないでください。

>[!NOTE]
>
>実験の開始後に実験の定義が変わると、実験は即座に無効になります。

1. Webまたはアプリケーションサーバーに管理者アクセス権を持っている場合は、Webクラスター内の任意の [!DNL Sensor] マシン上のインストール [!DNL Sensor] フォルダーに移動して、ファイルにアクセス [!DNL TestExperiment.xls] します。 管理者アクセス権がない場合は、アドビのアカウントマネージャーに問い合わせて、ファイルをリクエストし [!DNL TestExperiment.xls] てください。

## 設定ファイルとテストコンテンツのデプロイ {#section-34ff29649f584b93bc6129b75084b37c}

テストの設定ファイルは、Webクラスター内でテストを実行し、テストに関わるページを提供する各マ [!DNL Sensor] シンに展開する必要があります。 これは、手動の手順または既存のコンテンツ管理システムを使用して行うことができます。

**テストコンテンツを導入するには**

* テストに関わるページを提供するアプリケーショ [!DNL Sensor] ンまたはWebサーバーを実行するたびに、既存の公開プロセスを使用して、テストコンテンツを適切な場所に配置します。

   例えば、テストグループページをWebサイト( [!DNL index2.asp] )のテストフォルダーに公開す [!DNL mysite.com]る場合は、にファイルを公開します [!DNL www.mysite.com/test]。

   >[!NOTE]
   >
   >Webサイト上のページから直接テストファイルにリンクしないでください。 これにより、テスト結果とインデックススコアが無効になります。

**テストを導入するには**

* テストに関わるページを提供する [!DNL Sensor] Appを実行している各アプリケーションまたはWebサーバーで、設定ファイルのExpFileパラメータで指定したディレクトリにテスト設定テキストファイルを配置 [!DNL Sensor] します。 ExpFileパラ [メータの変更を参照](../../../home/c-undst-ctrld-exp/t-en-ctrld-exp/c-mod-expfile-prm.md#concept-25232b386a654870becc789d4f1fcc28)。

[!DNL Sensor] ファイル内で定義した割合に基づいて、各グループのWebサイト訪問者をランダムに選択し、必要に応じてテストまたはグループのコンテンツを提供します。
