# 実験設計スプレッドシート{#experiment-design-spreadsheet}

このファイルは、ワークシートとしてだけでなく、テストに関する意思決定の記録としても機能します。

テストの設計に役立つ場合は、Adobeが提供するテスト設計スプレッドシート（デフォルトではVS Controlled Experient Design.xlsという名前）を使用できます。

テストデザインスプレッドシートでは、対象の指標が特定の条件を満たす訪問者の割合として定義されている場合にのみ、有用な統計的推論を提供できます。 つまり、訪問者ベースの指標仮説をテストする場合にのみ役立ちます。

**テスト設計ファイルを使用してテストを設計するには**

1. Webサーバーまたはアプリケーションサーバーに管理者アクセス権を持っている場合は、Webクラスター内の任意の[!DNL Sensor]マシンの[!DNL Sensor]インストールフォルダーに移動します。 管理者アクセス権がない場合は、Adobeのアカウントマネージャーに問い合わせて、ファイルを要求してください。
1. VS Controlled Experient Design.xlsファイルを開きます。 （必要に応じて、このファイルの名前を変更できます）。

   以下のページのスプレッドシートは、本ガイド全体で使用するサンプル仮説をテストする準備を行う際のスプレッドシートの完成例です。

   ![](assets/experimentdesigntop.png)

   ![](assets/experimentdesignmiddle.png)

   ![](assets/experimentdesignbottom.png)

残りのフィールドは入力した値に基づいて計算され、次の表で説明します。

| フィールド | 説明 |
|---|---|
| アプリ選択の期待指標 | Webサイトに対して予想される指標値の見出しです。 |
| 期間ごとの予想訪問者数 | このフィールドは通常、スプレッドシートで自動的に計算されます。 ほとんどの日に、Webサイトがリターン訪問者よりも多くの新しい訪問者を受け取るという想定に依存しています。 そうでない場合は、このセルの計算は、テスト中に予想される実際の訪問者数で上書きされます。 |
| タイプIエラーの計算されたZスコア | 偽陽性の結果のZスコアです。 これは中間の統計計算です。 |
| タイプIIエラーの計算されたZスコア | 偽陰性の結果のZスコアです。 これは中間の統計計算です。 |
| 必要な最小訪問者数(Test+コントロール母集団) | 指定した信頼水準、電力レベルおよびZスコアを満たすためにテストに必要な最小訪問者数。「期間ごとの予測訪問者数」フィールドの値に対する割合で表されます。 |
| 必要な最小訪問者数(Test+コントロール母集団) | 指定した信頼水準、電力レベル、Zスコアを満たすためにテストに必要な最小訪問者数。 この値は、「訪問者」セクションの「合計（通常100%）」フィールドの値以下にする必要があります。 |
| テストの最小時間（日） | 指定した信頼水準、電力レベル、Zスコアを満たすためにテストを実行する必要がある最小日数。 この計算された数値は、「期間ごとの予測訪問者数」フィールドで説明したのと同じ問題になります。 再訪問者の多いWebサイトの場合、「Minimum Experiment Time (Days)」フィールドは、個別訪問者の数が「Minimum Experted」フィールドの値と等しくなるまでに予想される日数です。 |
| 訪問者 | 訪問者値の見出しです。 |
| テストグループ | テストグループで必要な訪問者の数。 |
| コントロール母集団 | コントロール母集団で必要な訪問者の数。 |
| 合計（通常100%） | テストに必要な訪問者の合計数。 この値は、「必須の最小訪問者数(テスト+コントロール母集団)」フィールドの値以上である必要があります。 |
| テストグループの精度(ターゲット信頼性レベル) | テストグループに対して計算された指標の測定値が、その真の値のこの割合内にあるという確率が、指定した信頼水準と等しい可能性があることを示す割合。 |
| コントロール母集団の正確性(ターゲット信頼性レベル) | コントロール母集団に対して計算された指標の測定値が、その真の値のこの割合内にあるという確率が、指定した信頼水準と等しい可能性があることを示す割合。 |
| zスコア(ターゲット精度) | 指定した値の標準偏差の数は、テストの平均値から来ています。 |
| 実際の信頼性レベル(ターゲット間隔) | テストで達成した信頼水準。 信頼水準は、指定された期待値が真となる確率を測定します。 |
| 実際の間隔(ターゲットの信頼性レベル) | テストで得られる信頼区間。未知の母集団パラメーターを含むと考えられる値の予測範囲を提供します。 この範囲は、指定されたサンプルデータのセットから計算されます。 |

「必須訪問者(テスト+コントロール母集団)」フィールドの値を確認する必要があります。..

![](assets/Experiment_Design_Min_Visitors.png)

[!DNL Visitors]列の合計フィールドの値と比較します。

![](assets/Experiment_Design_Total_Visitors.png)

テストが統計的に有効であるためには、「合計（通常100%）」フィールドの値が「最小訪問者必須(テスト+コントロール母集団)」フィールドの値と等しいかそれ以上である必要があります。

入力を見ると、例のワークシートで示すように、10,475人の訪問者がこの実験に参加し、入力された95%の信頼性率（この値を増やすことはできますが、任意の対照実験で最低推奨される信頼性）を達成する必要があります。 現在設計されているテストには30,000個の訪問者が含まれ、これは必要な最小限の訪問者数を大きく超えています。

日数を同じに維持する場合、訪問者の合計数が必要な最小値を満たし続けるかそれ以上になる限り、信頼性レベルを増やすことができます。

1. レコード用にファイルを保存し、そのファイルの情報を使用してテスト設定スプレッドシートを使用してテストを設定します。 このスプレッドシートの詳細については、「[テストの設定と導入](../../home/c-undst-ctrld-exp/t-crt-ctrld-exp/c-cnfg-dply-exp.md#concept-50f1de0242904698937bb72b3ea1b429)」を参照してください。
